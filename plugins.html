Repositório de módulos para QGIS

Desde Agosto 2014 o Grupo utilizadores QGIS PT disponibiliza um repositório de módulos (plugins) para QGIS onde irão ser publicadas ferramentas criadas pelo grupo, para o uso no contexto das tarefas/dados específicas/os de Portugal.
O repositório encontra-se no seguinte URL:

http://www.qgis.pt/plugins/plugins.xml


Para acrescentar o repositório no QGIS siga as seguintes opções:

(Inglês): Plugins -> Manage and Install Plugins -> Settings -> Plugins Repositories -> Add...

(português): Módulos -> Gerir e Instalar Módulos -> Configurações -> Repositório de Módulos -> Adicionar...

Transformação de Datum para Portugal

Este módulo foi criado para faciliar a transformação de dados vectoriais e raster em sistemas de coordenadas obsoletos (Datum 73, Datum Lisboa e ED50) para o novo sistema de coordenadas oficial de Portugal Continental (ETRS89-PTTM06), utilizando o método das grelhas de transformação de Datum (NTv2).

As grelha suportadas são as disponibilizadas pelo Prof. José Alberto Gonçalves (UP) e pela Direção-Geral do Território (DGT):

http://www.fc.up.pt/pessoas/jagoncal/coordenadas/

http://www.dgterritorio.pt/cartografia_e_geodesia/geodesia/transformacao_de_coordenadas/grelhas_em_ntv2/


As grelhas são disponibilizadas/instaladas em conjunto com a módulo e portanto não é pedida nenhuma configuração manual por parte do utilizador. O módulo criado tem como objectivo simplificar a tarefa do utilizador. Notas:

  • O desenvolvimento do módulo foi suportado pelo Grupo utilizadores QGIS PT e realizado pelo mesmo grupo, pela Faunalia e Alexander Bruy
  • Como qualquer outra ferramenta da toolbox "Processamento" de QGIS, também estas ferramentas podem funcionar em modalidade batch, tornando muito simples a tarefa da transformação de coordenadas de conjuntos de dados grandes
  • Repositório do código fonte: https://github.com/qgispt/processing_pttransform
  • Bug tracker: https://github.com/qgispt/processing_pttransform/issues
  • O módulo requer QGIS versão 2.4 ou master (a próxima versão 2.6) para funcionar.
  • É necessário verificar que na pasta .qgis2/python/plugins não seja presente uma pasta "processing". Se estiver presente é necessário apagá-la antes de instalar o módulo.

Problemas conhecidos:

  • Nas ferramentas de transformação de raster há um bug conhecido no QGIS 2.4, mas já resolvido no QGIS master: se a opção "abrir ficheiro de saída após execução do algoritmo" for activa então o raster será carregado com o sistema de coordenadas errado (o dos dados do ficheiro de entrada). Se o raster for carregado manualmente no QGIS este problema não se verifica.
  • Nalguns casos no QGIS 2.4 poderá aparecer um aviso (não é um erro!) sem razão aparente: "As camadas não estão no mesmo SRC. Isto pode causar resultados inesperados. Deseja continuar?". O aviso pode-se ignorar e de qualquer forma o problema já foi resolvido no QGIS master.
  • Os caminhos dos ficheiros de entrada/saída não devem conter caracteres especiais (espaços, acentos, tiles, etc.).
  • Actualmente as ferramentas de transformação de vectores funcionam apenas com ficheiros de entrada no formato Shapefile.
